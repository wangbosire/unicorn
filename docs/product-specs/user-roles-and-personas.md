# 用户角色与权限视角

本文档从产品规格视角单独整理后台角色、C 端角色与对应能力边界，便于需求澄清、页面设计与权限讨论时快速引用。

## 适用范围

- 后台菜单、操作按钮、列表权限与审核职责讨论
- 小程序端用户任务流与可见能力讨论
- 产品规格、设计文档与执行计划之间的角色口径对齐

## 后台角色

- 超级管理员：拥有所有后台权限。
- 发行运营：负责系列、批次、激活码管理。
- 内容审核员：负责藏品内容人工复核。
- 评论审核员：负责评论审核。
- 会员运营/客服：负责会员查看、冻结解冻、资产与行为查询。

## C 端角色

- 未登录访客：可浏览已公开的藏品展示页，不可评论。
- 登录会员：可激活藏品、管理自己名下藏品内容、评论、转让、查看消息。
- 藏品拥有者：对自己名下藏品具有编辑、提交审核、发起转让等权限。

## 角色使用说明

- 后台角色用于区分管理职责和后台权限，不等同于会员身份
- C 端角色用于区分访问状态与资源操作能力，不走后台 RBAC
- 藏品拥有者是登录会员中的资源归属子集，关键能力由“是否拥有该藏品”决定
