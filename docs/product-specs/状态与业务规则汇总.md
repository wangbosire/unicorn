## 权限分组

- 系列管理
- 批次管理
- 激活码管理
- 藏品管理
- 内容复核
- 评论审核
- 会员管理
- 转让记录
- 通知中心
- 系统管理

说明：

- 后台权限采用 RBAC。
- 一期至少区分页面权限和动作权限两层。

## 状态口径

- 激活码：`未发放 -> 已发放 -> 已使用`，异常或终态补充 `已作废`、`已失效`
- 藏品：`待领取 -> 已领取`，治理状态补充 `冻结`、`作废`
- 内容公开：`草稿 -> 已公开`，治理终态补充 `已下架`
- 内容审核：`待机审 -> 机审通过或机审失败 -> 待人工复核 -> 人工通过或人工驳回`
- 评论：`待机审 -> 机审通过或机审失败 -> 待人工审核 -> 人工通过或人工驳回`，治理终态补充 `已屏蔽`
- 转让单：`待接收 -> 已完成`，异常或终态补充 `已取消`、`已失效`
- 会员：`正常`、`冻结`

## 关键规则

### 账号与权限边界

- 后台用户体系与会员体系完全独立。
- 会员不支持后台手工创建。
- 后台能力必须按角色和动作拆分授权。

### 发行与领取

- 生成激活码时同步生成藏品。
- 激活成功后，藏品归属到当前会员。
- 藏品编号由系统自动生成。
- 系列与发行批次只能由后台配置。
- 批次停用后，已生成且仍有效的激活码可以继续激活。

### 内容与审核

- 内容审核记录必须完整保留。
- 新版本未审核通过前，旧公开版本继续在线。
- 冻结会员不会自动下架其已公开内容。
- 评论一期只做评论与审核，不做点赞和收藏。

### 转让与公开展示

- 公开页地址在转让后保持不变。
- 有草稿或待审核内容的藏品不可发起转让。
- 同一藏品同一时间只能存在一个有效转让单。

### 通知与可追溯性

- 站内信和站外通知都需要支持。
- 审核、下架、转让、激活等关键动作都应保留记录，便于客服、运营和排查使用。

## 维护规则
