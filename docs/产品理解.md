## 当前判断

- 一期主链路已经完成收口，回看产品资料时仍以“发码 -> 领藏品 -> 编辑内容 -> 审核 -> 展示”为主线理解能力边界
- 已归档的一期文档默认不继续吸纳二期设想；新增产品方向应拆分到新的专题或执行计划中
- 页面数量不是目标，规则边界清晰、体验口径一致、后续扩展不破坏已有闭环才是当前重点

## 维护规则

- 产品入口页只保留主线和阅读顺序
- 业务规则、流程、角色差异进入 `product-specs/` 专题页
