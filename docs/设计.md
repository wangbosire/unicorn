# 设计

`docs/design-docs/` 用于沉淀系统设计、架构约束、领域模型、接口规范与工程设计决策。

## 写作原则

- 默认按“AI 先读”的方式写：先结论、后展开
- 每篇文档优先聚焦单一主题，避免跨主题重复解释
- 入口与索引只回答“看什么、什么时候看”，不承担正文
- 设计稳定后优先回填到短文档或专题文档，而不是继续堆大杂烩
