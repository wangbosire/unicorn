# QUALITY_SCORE

本文档用于定义文档与工程质量评估方式，并持续追踪当前仓库在知识记录系统上的差距。

## 评分方法

- 采用 `0-5` 分制，`0` 表示缺失，`5` 表示稳定、可复用、可持续维护
- 每个维度都应给出当前分、证据入口、主要缺口与下一步动作
- 评分对象优先覆盖产品规格、设计文档、执行计划、生成文档、参考资料五类知识资产

## 分数定义

- `0`：缺失，仓库内没有有效材料
- `1`：仅有目录或占位文件，几乎不可用
- `2`：已有内容，但结构松散、缺少索引或维护约定
- `3`：有稳定入口和核心内容，可支撑当前阶段协作
- `4`：内容较完整，能支撑跨角色与智能体稳定复用
- `5`：高质量、持续更新、可校验，并与实现和计划保持同步
