# docs 索引

`docs/` 用来回答三类问题：产品要做什么、系统怎么设计、阶段工作做到哪。

## 先看哪里

- 想快速了解全局：先看 [产品理解.md](./产品理解.md)、[设计.md](./设计.md)、[执行计划.md](./执行计划.md)
- 想看产品规则：看 [product-specs/索引.md](./product-specs/索引.md)
- 想看系统设计：看 [design-docs/索引.md](./design-docs/索引.md)
- 想看阶段计划与归档：看 [exec-plans/active/二期-通知与转让运营增强.md](./exec-plans/active/二期-通知与转让运营增强.md) 和 [exec-plans/completed/一期V1归档总览.md](./exec-plans/completed/一期V1归档总览.md)
- 想看生成材料或外部资料：看 [generated/索引.md](./generated/索引.md) 和 [references/索引.md](./references/索引.md)

## 目录职责

- `product-specs/`：产品目标、角色、流程、业务规则
- `design-docs/`：架构、模型、状态、接口、工程规范
- `exec-plans/`：当前计划与阶段归档总览
- `generated/`：由源码、脚本或模型派生出的辅助文档
- `references/`：外部资料的项目化转译版本

## 当前口径

- 一期主链路已完成收口
- 当前文档重点是收口稳定结论、保留可追溯归档、为后续阶段提供清晰基线

## 维护规则

- 入口页只做导航和摘要，不复制正文
- 稳定规则写入专题文档，不留在聊天记录
- 新增专题优先补对应目录的 `索引.md` 或总入口链接
