# 一期V1归档总览

## 文档目标

集中保留一期 `V1` 主链路、`M1` 到 `M4` 里程碑以及评论治理专项的最终结论，作为当前仓库最重要的实施历史总览。

## 归档状态

- 状态：`completed`
- 归档范围：一期 `V1` 上位计划、`M1-M4` 里程碑、评论治理专项
- 归档时间基线：截至 `2026-05-18`

## 快速摘要

- 一期已经完成“发码 -> 激活 -> 编辑内容 -> 审核 -> 展示 -> 评论 -> 转让 -> 通知”的业务闭环。
- 后台已完成发行、审核治理、会员治理、通知总览、转让记录、仪表盘等运营主链路。
- 小程序已完成登录、激活、我的藏品、内容编辑、公开展示、评论、消息与基础转让链路。
- 当前若要理解“一期到底交付了什么”，只需要读本页，不必再翻多个阶段清单。

## 一期交付结论

### `apps/admin`

- 后台登录与 RBAC 基础能力
- 系列、批次、激活码、藏品与会员治理
- 内容审核、评论审核、通知总览、转让记录、仪表盘

### `apps/miniapp`

- 微信登录与会员身份确认
- 激活我的藏品、我的藏品、内容编辑、公开展示
- 评论交互、消息查看、基础转让链路

### `apps/api`

- `admin-api`、`member-api`、`public-api` 三类入口的一期主链路闭环
- 会员正式 Bearer 鉴权
- 内容审核、评论治理、转让记录、通知总览相关接口

## 里程碑摘要

### M1 最小发行闭环

- 完成后台登录、系列/批次、激活码生成、会员激活并领取藏品。
- 核心意义：打通从发行到资产归属的最小闭环。

### M2 内容展示闭环

- 完成会员内容编辑、保存草稿、提交审核、公开展示读取。
- 核心意义：让“领到藏品后可以编辑并展示内容”成为真实可用链路。

### M3 审核治理闭环

- 完成内容审核、人工复核、会员治理、评论治理相关基础能力。
- 核心意义：让一期具备基础风控和运营治理能力，而不只是展示能力。

### M4 流转与通知闭环

- 完成转让记录、通知总览、仪表盘统计与正式会员鉴权联调。
- 核心意义：让后台运营闭环与会员正式身份体系完成收口。

### 评论治理专项

- 完成评论模型、评论审核状态、三类 API、后台治理页面和专项回归入口。
- 核心意义：把评论从“范围内待实现”收口为独立可回归能力。

## 一期未纳入范围

- 通知模板管理器和异步派发 worker
- 更完整的会员端转让撤销、补偿和运营审计
- 点赞、收藏、举报等增强互动能力
- 更深的内容编辑器和展示模板
