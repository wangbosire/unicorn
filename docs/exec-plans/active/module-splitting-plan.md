# 模块拆分建议

本文档从一期执行计划中抽出模块拆分视图，便于研发启动、任务拆分和并行开发规划时快速引用。

## 当前状态

- 状态：`active`
- 上位计划：[V1 执行计划](./v1-exec-plan.md)

## 后端优先模块

- 会员与授权绑定
- RBAC 与后台账户
- 系列/批次/激活码/藏品模型
- 藏品内容版本模型
- 审核状态机
- 评论模型
- 转让单模型
- 通知事件模型

## 前端优先模块

- 后台登录与权限框架
- 系列/批次/激活码基础页面
- 藏品列表与详情
- C 端登录
- 激活我的藏品
- 我的藏品
- 内容编辑页
- 公开展示页

## 当前建议

- M1 已通过验收；当前以 **M2（内容展示闭环）** 为第一实现与验收目标，尽快打通「编辑 → 机审 → 公开展示」。
- M3 再补强审核治理、评论、会员运营。
- M4 最后完成转让、通知与仪表盘。

## 风险提醒

- 微信双渠道登录归并规则需尽早确定并联调验证。
- 内容审核状态与发布状态必须分离，否则后续人工复核会混乱。
- 转让功能依赖藏品归属与内容版本模型稳定，不建议提前开发。
- 通知中心要建立统一事件模型，避免后期各模块重复造消息逻辑。
