# M2 验收清单

## 文档目标

本文档用于定义 M2「内容展示闭环」的验收范围、验收条件、演示脚本与出入口标准，与 [一期里程碑总览](./milestones-overview.md)、[V1 执行计划](./v1-exec-plan.md) 对齐，作为 **M1 已完成** 后的阶段验收基线。

## 当前状态

- 状态：`active`（随实现进展补充演示步骤与数据核对项）
- 上位计划：[V1 执行计划](./v1-exec-plan.md)
- 前置里程碑：M1 已通过（见 [M1 验收清单（已归档）](../completed/m1-acceptance-checklist.md)）

## 目标结果

- 统一 M2 闭环的范围边界（与 M3「人工治理」区分）
- 统一「通过验收」的判断标准
- 为后端测试、小程序与公开展示联调提供同一套检查口径

## 范围内事项

- 藏品内容版本模型在会员侧可创建/迭代（至少首版草稿）
- 内容编辑基础能力（字段以设计与契约为准）
- 保存草稿、提交审核
- **机审**结果驱动公开可见性（通过则公开展示；失败则不公开；若有「疑似」策略以设计文档为准）
- **公开展示页**（匿名或可分享入口，以设计为准）可读到已发布内容

## 范围外事项（划入 M3 或后续）

- 人工复核工作台、下架兜底页、审核历史完整运营能力（见 M3）
- 评论体系、转让、通知中心
- 微信小程序正式登录态若尚未就绪，可在清单「联调前提」中声明 mock 边界（与 M1 一致做法）

## M2 范围摘要

对齐主链路：**领藏品 → 编辑内容 → 提交审核 → 机审 → 公开展示** 中与「编辑 + 机审公开 + 展示」相关的最小闭环。

## 依赖关系

- 依赖 M1：会员上下文、藏品归属、`OWNED` 等前置状态稳定
- 内容版本、编辑状态、发布状态、审核状态字段需与设计文档一致，避免与 M3 人工流混淆
- 机审策略（同步/异步、回调、失败文案）需有明确契约与错误码

## 验收结论标准

以下三类同时满足时，M2 可视为通过：

- 主路径流程可走通（编辑 → 提交 → 机审通过 → 公开展示可读）
- 核心状态与数据正确（版本号、发布态、可见性）
- 关键失败路径可控（校验失败、机审拒绝、未登录/无权限等）

## 业务流程验收项（初稿）

### 1. 内容与版本

- 已拥有藏品的会员可进入内容编辑入口
- 可保存草稿并可再次打开继续编辑（若本期支持）
- 提交审核后进入待审/机审中状态（具体枚举以实现为准）

### 2. 机审与公开

- 机审**通过**后，公开展示页可访问且内容与提交一致（字段范围以契约为准）
- 机审**不通过**时，公开展示页不可访问或不展示违规内容，且不产生错误归属

### 3. 公开展示页

- 通过机审的内容可通过约定 URL 或入口访问
- 未发布/未通过机审的内容不可被公开访问（含直接猜 ID 等场景，以实现为准）

## 异常路径验收项（初稿）

- 未登录访问会员专属编辑接口应被拒绝
- 非藏品所有者编辑应被拒绝
- 机审失败、重复提交、非法状态流转等返回明确错误且不损坏已有版本数据

## 出入口标准

### M2 入口条件

- M1 验收已通过
- 内容版本与审核相关枚举、接口契约已评审

### M2 出口条件

- 主链路演示可一次走通
- 核心接口与状态校验有测试覆盖（成功路径 + 若干关键失败路径）
- 与 M3 的边界（人工复核、下架）在文档与代码注释中可区分

## 建议测试覆盖（初稿）

### 后端

- 草稿保存/提交审核
- 机审通过/失败分支
- 公开展示读取接口（权限与可见性）

### 前端

- 小程序（或 H5）内容编辑与提交流程
- 公开展示页渲染与错误态
