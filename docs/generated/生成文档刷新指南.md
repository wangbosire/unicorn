# 生成文档刷新指南

## 文档目标

说明当前 `generated/` 目录下各类产物的来源和刷新方式。

## 总规则

- 生成文档不是源码真相源
- 生成文档更新时，应同时写明来源和刷新步骤
- 没有自动脚本时，也要保留最小人工刷新流程

## 当前产物

### `数据库Schema摘要.md`

- 来源：`apps/api/prisma/schema.prisma`
- 用途：快速查看当前枚举、模型覆盖范围和缺失业务域
- 刷新时机：Prisma schema 发生结构性变化后

手工刷新步骤：

1. 确认 `apps/api/prisma/schema.prisma` 已是最新版本
2. 用 `rg '^(enum|model) ' apps/api/prisma/schema.prisma` 快速列出当前枚举与模型
3. 对照 schema 更新 `数据库Schema摘要.md` 中的“枚举总览”“模型覆盖”“暂未覆盖”
4. 若新增业务域，补充对应覆盖判断，但不要复制完整字段树

### `mobile-ui-v1/`

- 来源：移动端 UI 稿导出图片与配套说明
- 用途：保留小程序主链路首轮视觉方向
- 刷新时机：页面结构或视觉方向出现明显调整后

手工刷新步骤：

1. 替换导出的页面图片
2. 更新 `mobile-ui-v1/说明.md` 中的范围、页面职责和后续建议
3. 若页面职责变化，确保与 `product-specs/` 和 `design-docs/` 口径一致

## 后续建议

- 若 `数据库Schema摘要.md` 更新频率持续升高，补成脚本生成
- 若后续新增 API/状态机导出，也应在本文补充来源和刷新步骤
