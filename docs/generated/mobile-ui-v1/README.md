# 移动端 UI 稿 V1

本文档用于承接一期移动端主链路的首轮 UI 稿输出，当前只覆盖会员端，不包含后台管理系统。

## 本轮范围

- 目标：围绕 `激活 -> 我的藏品 -> 编辑 -> 展示` 主链路，先收敛移动端页面结构和视觉方向
- 当前已出高保真稿：
  - 首页入口：[WRo8z.png](./WRo8z.png)
  - 激活页主体：[KC5fX.png](./KC5fX.png)
  - 我的藏品列表：[nTsDC.png](./nTsDC.png)
- 当前未展开完整高保真稿，但已明确结构方向：
  - 藏品内容编辑页
  - 公开展示页
  - 消息中心
  - 个人中心

## 视觉方向

- 气质：温暖、克制、带一点收藏品陈列感，不做强科技蓝或运营后台风
- 主色：深墨蓝作为主信息区，鼠尾草绿和暖米色作为状态区分
- 结构：移动端单列卡片流，强调“当前状态 + 下一步动作”
- 场景重点：不是做内容社区首页，而是帮助会员快速完成领取、补内容、提交审核

## 页面职责

### 1. 首页入口

- 作用：把“今天应该先做什么”说清楚
- 重点模块：
  - 主链路引导 Hero
  - 待激活 / 待继续 两个快捷入口
  - 本周进度
  - 最近领取藏品

### 2. 激活页主体

- 作用：承接激活码输入和激活成功预期
- 重点模块：
  - 激活说明
  - 激活码输入区
  - 激活主按钮
  - 成功结果预览
  - 统一异常提示区

### 3. 我的藏品列表

- 作用：按状态组织会员已拥有藏品，优先推动草稿和审核中内容继续前进
- 重点模块：
  - 状态筛选标签
  - 草稿中卡片
  - 审核中卡片
  - 已公开卡片
  - 卡片级动作：继续编辑、预览展示页

## 后续出稿建议

1. 先补“内容编辑页”和“公开展示页”，因为这两页直接决定 M2 的讨论效率。
2. 再补“消息中心”和“个人中心”，它们更适合在主链路稳定后承接通知和账户管理。
3. 如果要进入开发，建议先把这套视觉语言抽成小程序端的页面模板：
   - 深色 Hero
   - 状态卡片
   - 胶囊筛选标签
   - 底部主 CTA
