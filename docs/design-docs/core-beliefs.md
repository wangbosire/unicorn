# Core Beliefs

本文档用于沉淀仓库在产品设计、系统设计与工程推进上的核心信念，帮助团队在文档和实现中保持一致判断。

## 核心信念

- 先完成业务主链路闭环，再扩展外围能力
- 稳定规则必须文档化，不能只存在于口头约定
- 复杂状态与权限规则优先收口在后端与共享契约
- 设计文档应服务落地，不追求脱离执行的“空架构”
- 计划、设计、实现、验收必须使用同一套业务语言

## 当前适用范围

- `product-specs/` 中的需求范围判断
- `design-docs/` 中的架构、模型、接口与状态设计
- `exec-plans/` 中的优先级、里程碑与执行顺序决策
