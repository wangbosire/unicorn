# 数字藏品运营与展示平台审核与通知事件设计

## 文档目标

只保留关键事件分类、主要副作用和一期实现口径。

## 真相源

- 事件实现：`apps/api/src/modules/**`
- 事件名常量：`packages/shared-config/src/**`
- 通知与记录模型：`apps/api/prisma/schema.prisma`

## 总规则

- 主业务流程由应用服务编排
- 跨模块副作用优先通过事件解耦
- 事件名表达“已发生的业务事实”
- 通知失败不回滚主业务，但必须留痕

## 一期关键事件

### 发行与激活

- `issuance.activation_code.generated`
- `collection.created_for_activation`
- `member.collection.activated`

副作用：

- 记录审计日志
- 发送激活成功通知

### 内容编辑与审核

- `collection.content.draft_saved`
- `collection.content.submitted`
- `collection.content.machine_approved`
- `collection.content.machine_rejected`
- `collection.content.manual_review_requested`
- `collection.content.manual_approved`
- `collection.content.manual_rejected`
- `collection.content.published`
- `collection.content.taken_down`

副作用：

- 驱动机审
- 写入审核记录
- 更新公开页
- 发送审核结果通知

### 评论审核

- `comment.submitted`
- `comment.machine_approved`
- `comment.machine_rejected`
- `comment.manual_review_requested`
- `comment.manual_approved`
- `comment.manual_rejected`
- `comment.blocked`

### 转让

- `transfer.created`
- `transfer.accepted`
- `transfer.completed`
- `transfer.cancelled`
- `transfer.expired`

副作用：

- 更新藏品归属
- 写入转让记录
- 发送转让结果通知

## 一期通知类型

- 激活成功
- 内容审核通过 / 驳回
- 评论审核结果
- 转让待接收 / 已完成

## 通知渠道

- 站内信：默认必达
- 小程序订阅消息：按授权发送
- 公众号通知：按绑定情况发送

## 审计要求

以下动作必须可追溯：

- 激活码生成
- 藏品激活
- 内容提交审核
- 人工审核通过 / 驳回
- 评论审核通过 / 驳回
- 转让完成

## 一期实现口径

- 先使用进程内事件总线
- 事件统一包含 `eventName`、`occurredAt`、`operator`、`payload`
- 后续需要异步重试时再演进到消息队列
