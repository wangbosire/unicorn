# 数字藏品运营与展示平台一期数据模型草案

## 文档目标

用最短篇幅回答“系统里有哪些核心对象，它们之间是什么关系”。

## 真相源

- 关系与字段最终以 `apps/api/prisma/schema.prisma` 为准
- 本文只保留实体职责、关键关系和建模判断

## 建模规则

- 一个实体只承担一类核心职责
- 状态字段优先结构化，不用自由文本表达业务流程
- 历史、审核、派发、转让等过程优先单独建记录表
- 后台身份、会员身份、公开访问三条边界不混用

## 核心实体

### 后台身份域

- `AdminUser`：后台账号
- `Role`：角色
- `Permission`：权限点
- 作用：支撑后台 RBAC

### 会员域

- `Member`：会员主档
- `MemberWechatBinding`：微信渠道绑定
- 作用：统一会员身份，不和后台账号共表

### 发行域

- `Series`：长期主题或系列
- `IssuanceBatch`：一次发行批次
- `ActivationCode`：领取入口
- 作用：完成“发码 -> 待领取藏品”链路

### 藏品域

- `Collection`：藏品主实体
- `CollectionContentVersion`：内容版本快照
- 作用：表达归属、展示内容和可追溯版本

### 审核域

- `CollectionContentReviewRecord`：内容审核记录
- 作用：分离审核历史与内容本体

### 评论域

- `CollectionComment`
- `CollectionCommentReviewRecord`
- 作用：承载公开互动与治理记录

### 转让域

- `CollectionTransferOrder`
- 作用：承载转让流程与历史

### 通知域

- `NotificationMessage`
- `NotificationDispatchRecord`
- 作用：承载站内消息与外部派发

## 关键关系

- 一个 `Member` 可拥有多个 `Collection`
- 一个 `Series` 下有多个 `IssuanceBatch`
- 一个 `IssuanceBatch` 下有多个 `ActivationCode`
- 一个 `ActivationCode` 对应一个待领取 `Collection`
- 一个 `Collection` 可有多个 `CollectionContentVersion`
- 一个内容版本可有多条审核记录
- 一个 `Collection` 可有多条评论、转让、通知相关记录

## 一期必须稳定的枚举

- 系列状态
- 批次状态
- 激活码状态
- 藏品状态
- 内容编辑状态
- 内容发布状态
- 内容审核状态
- 会员状态
