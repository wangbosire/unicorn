# 数字藏品运营与展示平台鉴权与权限设计

## 文档目标

只保留身份边界、权限判断方式和不能破的安全规则。

## 三类身份

- 后台用户：运营、审核、客服、系统管理员
- 会员用户：小程序或公众号登录会员
- 匿名访客：公开页浏览者

## 核心规则

### 后台用户

- 独立登录入口
- 独立账户表
- 使用 RBAC

### 会员用户

- 通过微信授权接入
- 统一归并到会员主账户
- 不使用后台 RBAC
- 通过“登录态 + 资源归属 + 资源状态”授权

### 匿名访客

- 无需登录
- 只能访问公开内容
- 不可看到审核、风控、隐私信息

## 后台权限模型

- 页面访问权限
- 动作操作权限

典型动作权限：

- `issuance.series.create`
- `issuance.activation-code.generate`
- `collection.review.approve`
- `collection.review.reject`
- `comment.review.approve`
- `member.freeze`

## 会员权限模型

会员侧不建角色树，优先判断：

- 是否已登录
- 是否为资源拥有者
- 当前状态是否允许操作

典型例子：

- 只有当前拥有者可编辑藏品
- 冻结会员不可继续评论或转让
- 转让中藏品不可再次发起转让

## 公开访问规则

公开接口只返回：

- 已公开内容
- 对外可展示评论
- 不含内部审核信息的展示字段

## 接口落点

- `admin-api`：后台身份 + RBAC
- `member-api`：会员身份 + 归属校验 + 状态校验
- `public-api`：匿名可访问，仅暴露公开数据

## 一期必须落实

- 后台用户与会员不得混用账户体系
- 所有审核动作必须记录操作人
- 所有后台修改类接口必须做动作级鉴权
- 会员侧编辑、评论、转让必须做归属校验
