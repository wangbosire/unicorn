# 数字藏品运营与展示平台 API 约定文档

## 文档目标

只保留跨接口都成立的规则，避免在每个接口文档里重复解释。

## 真相源

- 契约：`packages/api-contracts/src/**`
- 实现：`apps/api/src/modules/**`

## 边界规则

- `admin-api`：后台运营与治理
- `member-api`：会员登录态能力
- `public-api`：匿名公开读取

## 路径规则

- 资源名用复数、短名、业务名词
- 路径表达资源，不表达页面动作
- 过滤、分页、搜索优先放 query
- 审核、下架、接收等少量状态动作可用明确子路径

示例：

- `/admin-api/series`
- `/admin-api/issuance-batches/:batchId`
- `/member-api/my/collections/:collectionId/content/drafts`

## 返回规则

- 成功：`success + data`
- 失败：`success + error`
- 列表：`items + page + pageSize + total`

## 时间与编号规则

- 时间字段统一 ISO 8601 字符串
- 业务编号与数据库主键分离
- 对外展示优先业务编号，如 `seriesNo`、`batchNo`、`collectionNo`

## 错误码规则

- 错误码全大写，下划线分隔
- 优先表达业务原因，不只表达 HTTP 状态
- 同一业务状态冲突尽量复用同一错误码

## 校验规则

- DTO 层处理格式和必填校验
- Service 层处理业务状态校验
- 权限校验在 Guard / Policy / Service 中明确收口，不散落到前端

## 共享契约规则

- DTO 真相源在 `packages/api-contracts`
- 前后端都优先引用共享契约，不重复手写结构
- 文档只保留边界和约定，不复制整份 DTO

## 审计与幂等

- 审核、激活、转让、派发等关键动作优先可追溯
- 可能重复提交的动作要有幂等判断

## 当前项目落点

- 契约：`packages/api-contracts/src/**`
- 实现：`apps/api/src/modules/**`
- 边界设计：`admin-api`、`member-api`、`public-api`

## 维护规则

- 只有跨接口通用的规则才写进本文
- 资源级差异放回各自模块或契约文件
