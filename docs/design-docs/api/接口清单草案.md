# 数字藏品运营与展示平台接口清单草案

## 文档目标

列出每个 API 边界下有哪些资源，不在本文重复铺开 DTO。

## 真相源

- 契约目录：`packages/api-contracts/src/**`
- 路由实现：`apps/api/src/modules/**`

## 使用方式

- 想找某个能力归哪个接口边界：看本文
- 想看字段：读 `packages/api-contracts/src/**`
- 想看命名和响应约定：读 [API 约定文档](./接口约定.md)

## `admin-api`

- `auth`：后台登录、当前身份
- `dashboard`：仪表盘概览
- `series`：系列管理
- `issuance-batches`：发行批次管理
- `activation-codes`：激活码生成与查询
- `collections`：藏品列表、详情、运营状态
- `collection-reviews`：内容审核、审核历史、下架
- `collection-comments`：评论治理
- `members`：会员查询、冻结、解冻
- `notifications`：通知总览
- `transfers`：转让记录治理

## `member-api`

- `auth`：微信登录、当前会员信息
- `collection-activation`：激活并领取藏品
- `my/collections`：我的藏品、内容读取、草稿、提交审核
- `my/messages`：我的消息
- `my/transfers`：我的转让记录、发起转让、接收转让
- `collection-comments`：发表评论、回复评论

## `public-api`

- `collections`：公开展示页
- `collections/:slug/comments`：公开评论列表

## 里程碑最小范围

### M1

- 后台登录
- 系列管理
- 发行批次管理
- 激活码管理
- 藏品列表
- 会员登录
- 激活藏品
- 我的藏品列表

### M2

- 我的藏品内容读取
- 保存草稿
- 提交审核
- 公开展示页

### M3

- 内容审核队列
- 审核历史
- 人工通过 / 驳回 / 下架
- 会员治理
- 评论治理

### M4

- 仪表盘
- 通知总览
- 转让治理
- 会员消息与转让链路

## 维护规则

- 本文只保留资源级接口清单
- 新增接口先更新 `packages/api-contracts`
- 某资源下新增子接口时，优先补到对应资源项，而不是展开成长列表
