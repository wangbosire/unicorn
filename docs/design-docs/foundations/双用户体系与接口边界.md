# 双用户体系与 API 边界

## 文档目标

只回答“后台用户、会员、公开访问三套边界怎么隔离”。

## 真相源

- 边界实现：`apps/api/src/modules/**`
- 后台权限常量：`packages/shared-config/src/**`
- 会员与后台契约：`packages/api-contracts/src/**`

## 总规则

- 后台用户与会员用户分表
- 后台登录态与会员登录态分离
- 后台走 RBAC，会员走归属校验
- 公开接口只返回可展示数据

## 数据隔离

- 后台表不承载会员身份信息
- 会员表不承载后台角色菜单信息

## 认证隔离

- 后台与会员使用独立登录入口
- 后台 token 不可访问会员接口
- 会员 token 不可访问后台接口

## 权限隔离

- 后台：RBAC
- 会员：登录态 + 资源拥有者 + 状态限制
- 公开：匿名只读

## API 边界

### `admin-api`

- 面向后台管理端
- 可访问发行、审核、会员管理、系统管理

示例：

- `/admin-api/auth/login`
- `/admin-api/series`
- `/admin-api/issuance-batches`
- `/admin-api/activation-codes`
- `/admin-api/collections`
- `/admin-api/collection-reviews`
- `/admin-api/collection-comments`
- `/admin-api/members`

### `member-api`

- 面向登录会员
- 不走 RBAC
- 走资源归属校验

示例：

- `/member-api/auth/wechat-miniapp`
- `/member-api/auth/wechat-mp`
- `/member-api/collection-activation`
- `/member-api/my/collections`
- `/member-api/my/messages`
- `/member-api/my/transfers`
- `/member-api/collection-comments`

### `public-api`

- 面向公开访问
- 默认不要求登录
- 只返回已公开内容

示例：

- `/public-api/collections/:slug`
- `/public-api/collections/:slug/comments`
- `/public-api/collections/:slug/stats`
