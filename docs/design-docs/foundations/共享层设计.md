# 共享层设计

## 文档目标

只回答“什么应该进共享层，什么不该进”。

## 真相源

- 包结构：`packages/**`
- 契约目录：`packages/api-contracts/src/**`
- 共享类型：`packages/shared-types/src/**`
- 共享配置：`packages/shared-config/src/**`

## 共享层职责

### `api-contracts`

- 前后端共享接口契约
- 按 `admin`、`member`、`public` 分组

### `shared-types`

- 状态枚举
- 共享领域类型
- 通用响应类型

### `shared-config`

- 权限 key
- 路由常量
- 事件名
- 跨端通用常量

### `ui`

- 可复用设计 token
- 通用样式约定
- 少量真正跨端可复用组件

### `utils`

- 纯工具函数
- 不承载业务规则

## 抽取规则

- 只抽稳定、跨模块复用、语义明确的内容
- 业务规则优先留在后端模块
- 状态枚举、接口契约、权限 key 优先沉淀到共享层
- 页面结构、局部流程、临时兼容逻辑不要硬抽成共享包
