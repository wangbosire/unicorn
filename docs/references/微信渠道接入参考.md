# 微信渠道接入参考

## 文档目标

整理当前仓库与微信小程序、微信公众号接入直接相关的项目化结论，方便后续联调和能力补全。

## 当前仓库口径

- 会员登录支持微信小程序与微信公众号两种入口
- 两种渠道统一归并到同一会员主账号
- 当前会员鉴权以正式 Bearer access token 为准
- 小程序端是当前最主要的会员使用场景

## 当前实现结论

- `member-api/auth/wechat-miniapp`：小程序登录入口
- `member-api/auth/wechat-mp`：公众号登录入口
- 两种渠道共享会员归并和 JWT 签发主流程，仅渠道绑定类型不同
- 通知渠道区分站内信、小程序订阅消息、公众号通知

## 产品与设计上要保持的约束

- 后台账号与会员账号不混用
- 微信渠道只是登录与通知入口，不改变会员主身份模型
- 小程序主链路优先围绕“登录 -> 激活 -> 我的藏品 -> 编辑 -> 展示”
- 若后续接入真实微信能力，应保持现有接口路径和前端调用口径尽量不变

## 后续接入真实微信能力时优先确认

- `code2Session` 或等价换取会话能力的接入方式
- `unionid` 可用性与归并规则
- 小程序订阅消息模板与发送时机
- 公众号通知模板与绑定前提
- 分享公开页时的小程序与 H5 跳转策略
